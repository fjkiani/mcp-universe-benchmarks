# Unique Domain Approach - Using Existing MCP Servers

## Key Insight from Emma Lee

**We don't need NEW external APIs** - we use **existing MCP servers** in our system.

## What Makes a Domain Unique?

Not new servers, but:
1. **Novel problem domain** - Different from investments/grant/identity
2. **Complex multi-server orchestration** - Using servers in novel combinations
3. **Challenging reasoning tasks** - Tests capabilities beyond simple tool use
4. **Unique workflow patterns** - Different from standard business automation

## What Emma Lee Did Differently

She proposed **education_personalization** - which likely:
- Uses **existing servers** (google-search, google-sheets, calendar, email)
- But in a **unique domain** (education/learning)
- With **complex workflows** (learner onboarding, curriculum personalization)
- Testing **different capabilities** (pedagogical reasoning, adaptive learning)

## Our Unique Domain Options

### Option 1: Healthcare Patient Onboarding
**Servers:** calendar, email, pdf-generator, google-sheets, task-management
**Unique:** Medical compliance, appointment scheduling, document generation
**Challenge:** HIPAA considerations, multi-step patient workflows

### Option 2: Supply Chain Orchestration
**Servers:** google-search, google-sheets, email, calendar, task-management
**Unique:** Multi-party coordination, inventory tracking, delivery scheduling
**Challenge:** State management across multiple actors, deadline coordination

### Option 3: Legal Document Automation
**Servers:** google-search, pdf-generator, file-storage, email, calendar
**Unique:** Legal reasoning, contract generation, compliance checking
**Challenge:** Template customization, legal compliance, multi-document workflows

### Option 4: Event Planning & Coordination
**Servers:** calendar, email, google-sheets, google-search, task-management
**Unique:** Multi-vendor coordination, timeline management, resource allocation
**Challenge:** Complex scheduling, conflict resolution, vendor communication

## What Makes It "Not Standard"

1. **Domain-specific reasoning** - Not generic business automation
2. **Complex state management** - Multi-step workflows with dependencies
3. **Multi-party coordination** - Not just single-agent tasks
4. **Challenging pass rate** - 30-40% to test real capabilities

## The Real Question

**What domain/problem space would:**
- Use existing MCP servers
- Test novel capabilities (not just tool calling)
- Challenge models with complex reasoning
- Be different from investments/grant/identity/education

**Answer:** Pick a domain that requires **orchestration, reasoning, and coordination** beyond simple tool use.

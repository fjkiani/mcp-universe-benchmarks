### File Naming Conventions

**Task Files:**
```
Format: {category}_task_{number}.json
        └─────┬────┘ └──┬─┘ └──┬─┘
              │         │      └─── 4-digit zero-padded number
              │         └────────── Always "task"
              └──────────────────── Descriptive category

Examples:
✅ email_send_task_0001.json
✅ email_read_task_0001.json
✅ contact_mgmt_task_0001.json
✅ multi-server_task_0001.json

❌ task_1.json                (no category, not zero-padded)
❌ email_sending_0001.json    (missing "task")
❌ EmailSendTask0001.json     (wrong case)
```

**Evaluator Functions:**
```python
# In evaluators/functions.py

Format: @compare_func(name="{domain}.{function_name}")

Examples:
✅ @compare_func(name="email_automation.check_email_sent")
✅ @compare_func(name="email_automation.llm_as_a_judge")
✅ @compare_func(name="web_search.llm_as_a_judge")

❌ @compare_func(name="check_email_sent")  # Missing domain prefix
❌ @compare_func(name="CheckEmailSent")    # Wrong case
```

## 🚫 What NOT to Commit

### Never Commit These

```
❌ .env                              # API keys
❌ .venv/                            # Virtual environment
❌ __pycache__/                      # Python cache
❌ *.pyc                             # Compiled Python
❌ .DS_Store                         # macOS files
❌ .idea/                            # IDE config
❌ .vscode/settings.json             # IDE settings (local)
❌ node_modules/                     # If using JS
❌ *.log                             # Log files
❌ secrets.yaml                      # Any secrets file
❌ api_keys.txt                      # Any keys file
```

These are already in `.gitignore`, but be careful!

### Verify Before Committing

```bash
# Check what you're about to commit
git status

# Review changes
git diff

# Check for secrets (if you have a tool)
git secrets --scan

# Add only what you need
git add domains/your_domain/

# Verify again
git status
```

## 🆘 Structure Problems?

### Issue: "Domain not found"

**Check:**
```bash
# Does directory exist?
ls -la domains/

# Is your domain there?
ls -la domains/your_domain/

# If not, create it
mkdir -p domains/your_domain
```

### Issue: "Config file invalid"

**Check:**
```bash
# Validate YAML syntax
cat domains/your_domain/config.yaml | python -c "import yaml, sys; yaml.safe_load(sys.stdin)"

# Check for:
# - Tabs instead of spaces (use spaces!)
# - Missing colons
# - Incorrect indentation
```

### Issue: "Tasks not found"

**Check:**
```bash
# Does tasks/ directory exist?
ls -la domains/your_domain/tasks/

# Are there 50+ JSON files?
ls domains/your_domain/tasks/*.json | wc -l

# Are they valid JSON?
for f in domains/your_domain/tasks/*.json; do
    echo "Checking $f"
    python -m json.tool "$f" > /dev/null || echo "❌ Invalid JSON: $f"
done
```

### Issue: "Evaluator import failed"

**Check:**
```bash
# Does evaluators/ exist?
ls -la domains/your_domain/evaluators/

# Is __init__.py present?
ls -la domains/your_domain/evaluators/__init__.py

# Does it import functions?
cat domains/your_domain/evaluators/__init__.py
# Should contain: from .functions import *

# Are function names correct?
grep "@compare_func" domains/your_domain/evaluators/functions.py
```

## 💡 Pro Tips

1. **Copy web_search structure exactly**
   ```bash
   cp -r domains/web_search domains/your_domain
   # Then modify files
   ```

2. **Use tree to visualize**
   ```bash
   tree domains/your_domain -L 2
   ```

3. **Count your files**
   ```bash
   # Task files
   ls domains/your_domain/tasks/*.json | wc -l

   # All files
   find domains/your_domain -type f | wc -l
   ```

4. **Validate structure before tasks**
   - Create all directories first
   - Add empty template files
   - Run structure validation
   - Then fill in content

5. **Keep backups**
   ```bash
   # Backup before major changes
   tar -czf backup_$(date +%Y%m%d).tar.gz domains/your_domain/
   ```

## 💡 Tips for Reading Documentation

1. **Don't read everything at once**
   - Start with GETTING_STARTED.md
   - Read others as needed

2. **Use search (Cmd+F / Ctrl+F)**
   - All docs are searchable
   - Find specific topics quickly

3. **Bookmark useful sections**
   - You'll reference them often
   - Especially troubleshooting

4. **Follow the links**
   - Docs cross-reference each other
   - Links take you to related info

5. **Read code examples**
   - `domains/web_search/` is your friend
   - Real working code is best teacher

6. **Take notes**
   - Jot down questions
   - Track your progress
   - Note useful commands
